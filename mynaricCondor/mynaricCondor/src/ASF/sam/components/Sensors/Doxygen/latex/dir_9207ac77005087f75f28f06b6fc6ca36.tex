\doxysubsection{Detailed Description}
\hypertarget{README.md_autotoc_md43}{}\doxysubsection{\texorpdfstring{Plugin Overview}{Plugin Overview}}\label{README.md_autotoc_md43}
Plugin for integrating GNU GCov code coverage tool into Ceedling projects. Currently only designed for the gcov command (like LCOV for example). In the future we could configure this to work with other code coverage tools.

This plugin currently uses \href{https://www.gcovr.com/}{\texttt{ gcovr}} and / or \href{https://danielpalme.github.io/ReportGenerator/}{\texttt{ Report\+Generator}} as utilities to generate HTML, XML, JSON, or Text reports. The normal gcov plugin {\itshape must} be run first for these reports to generate.\hypertarget{README.md_autotoc_md44}{}\doxysubsubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md44}
gcovr can be installed via pip like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{pip\ install\ gcovr}

\end{DoxyCode}


Report\+Generator can be installed via .NET Core like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{dotnet\ tool\ install\ -\/g\ dotnet-\/reportgenerator-\/globaltool}

\end{DoxyCode}


It is not required to install both {\ttfamily gcovr} and {\ttfamily Report\+Generator}. Either utility may be installed to create reports.\hypertarget{README.md_autotoc_md45}{}\doxysubsubsection{\texorpdfstring{Configuration}{Configuration}}\label{README.md_autotoc_md45}
The gcov plugin supports configuration options via your {\ttfamily project.\+yml} provided by Ceedling.\hypertarget{README.md_autotoc_md46}{}\doxysubsubsubsection{\texorpdfstring{Utilities}{Utilities}}\label{README.md_autotoc_md46}

\hypertarget{README.md_autotoc_md53}{}\doxysubsubsubsection{\texorpdfstring{Common Report Options}{Common Report Options}}\label{README.md_autotoc_md53}
There are a number of options to control which files are considered part of the coverage report. Most often, we only care about coverage on our source code, and not on tests or automatically generated mocks, runners, etc. However, there are times where this isn\textquotesingle{}t true... or there are times where we\textquotesingle{}ve moved ceedling\textquotesingle{}s directory structure so that the project file isn\textquotesingle{}t at the root of the project anymore. In these cases, you may need to tweak {\ttfamily report\+\_\+include}, {\ttfamily report\+\_\+exclude}, and {\ttfamily exclude\+\_\+directories}.

One important note about {\ttfamily report\+\_\+root}\+: gcovr will take only a single root folder, unlike Ceedling\textquotesingle{}s ability to take as many as you like. So you will need to choose a folder which is a superset of ALL the folders you want, and then use the include or exclude options to set up patterns of files to pay attention to or ignore. It\textquotesingle{}s not ideal, but it works.

Finally, there are a number of settings which can be specified to adjust the default behaviors of gcovr\+:
