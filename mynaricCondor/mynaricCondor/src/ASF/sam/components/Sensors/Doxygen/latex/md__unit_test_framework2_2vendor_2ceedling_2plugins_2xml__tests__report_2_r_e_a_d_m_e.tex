The xml\+\_\+tests\+\_\+report plugin creates an XML file of test results in x\+Unit format, which is handy for Continuous Integration build servers or as input into other reporting tools. The XML file is output to the appropriate {\ttfamily \texorpdfstring{$<$}{<}build\+\_\+root\texorpdfstring{$>$}{>}/artifacts/} directory (e.\+g. {\ttfamily artifacts/test/} for test tasks, {\ttfamily artifacts/gcov/} for gcov, or {\ttfamily artifacts/bullseye/} for bullseye runs).\hypertarget{md__unit_test_framework2_2vendor_2ceedling_2plugins_2xml__tests__report_2_r_e_a_d_m_e_autotoc_md168}{}\doxysection{\texorpdfstring{Setup}{Setup}}\label{md__unit_test_framework2_2vendor_2ceedling_2plugins_2xml__tests__report_2_r_e_a_d_m_e_autotoc_md168}
