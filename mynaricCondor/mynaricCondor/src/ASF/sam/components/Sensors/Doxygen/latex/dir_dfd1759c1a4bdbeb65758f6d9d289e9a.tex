\doxysubsection{Detailed Description}
This is a plug-\/in for \href{https://github.com/ThrowTheSwitch/Ceedling}{\texttt{ Ceedling}} to use the \href{https://github.com/meekrosoft/fff}{\texttt{ Fake Function Framework}} for mocking instead of CMock.

Using fff provides less strict mocking than CMock, and allows for more loosely-\/coupled tests. And, when tests fail -- since you get the actual line number of the failure -- it\textquotesingle{}s a lot easier to figure out what went wrong.\hypertarget{README.md_autotoc_md27}{}\doxysubsection{\texorpdfstring{Installing the plug-\/in}{Installing the plug-\/in}}\label{README.md_autotoc_md27}
To use the plugin you need to 1) get the contents of this repo and 2) configure your project to use it.\hypertarget{README.md_autotoc_md28}{}\doxysubsubsection{\texorpdfstring{Get the source}{Get the source}}\label{README.md_autotoc_md28}
The easiest way to get the source is to just clone this repo into the Ceedling plugin folder for your existing Ceedling project. (Don\textquotesingle{}t have a Ceedling project already? \href{http://www.electronvector.com/blog/try-embedded-test-driven-development-right-now-with-ceedling}{\texttt{ Here are instructions to create one.}}) From within {\ttfamily \texorpdfstring{$<$}{<}your-\/project\texorpdfstring{$>$}{>}/vendor/ceedling/plugins}, run\+:

{\ttfamily git clone \href{https://github.com/ElectronVector/fake_function_framework.git}{\texttt{ https\+://github.\+com/\+Electron\+Vector/fake\+\_\+function\+\_\+framework.\+git}}}

This will create a new folder named {\ttfamily fake\+\_\+function\+\_\+framework} in the plugins folder.\hypertarget{README.md_autotoc_md29}{}\doxysubsubsection{\texorpdfstring{Enable the plug-\/in.}{Enable the plug-\/in.}}\label{README.md_autotoc_md29}
The plug-\/in is enabled from within your project.\+yml file.

In the {\ttfamily \+:plugins} configuration, add {\ttfamily fake\+\_\+function\+\_\+framework} to the list of enabled plugins\+:

In that case you\textquotesingle{}d need to add a new path the {\ttfamily \+:load\+\_\+paths}.}\hypertarget{README.md_autotoc_md30}{}\doxysubsection{\texorpdfstring{How to use it}{How to use it}}\label{README.md_autotoc_md30}
You use fff with Ceedling the same way you used to use CMock. Modules can still be generated with the default module generator\+: {\ttfamily rake module\+:create\mbox{[}my\+\_\+module\mbox{]}}. If you want to "{}mock"{} {\ttfamily some\+\_\+module.\+h} in your tests, just {\ttfamily \#include "{}mock\+\_\+some\+\_\+module.\+h"{}}. This creates a fake function for each of the functions defined in {\ttfamily some\+\_\+module.\+h}.

The name of each fake is the original function name with an appended {\ttfamily \+\_\+fake}. For example, if we\textquotesingle{}re generating fakes for a stack module with {\ttfamily push} and {\ttfamily pop} functions, we would have the fakes {\ttfamily push\+\_\+fake} and {\ttfamily pop\+\_\+fake}. These fakes are linked into our test executable so that any time our unit under test calls {\ttfamily push} or {\ttfamily pop} our fakes are called instead.

Each of these fakes is actually a structure containing information about how the function was called, and what it might return. We can use Unity to inspect these fakes in our tests, and verify the interactions of our units. There is also a global structure named {\ttfamily fff} which we can use to check the sequence of calls.

The fakes can also be configured to return particular values, so you can exercise the unit under test however you want.

The examples below explain how to use fff to test a variety of module interactions. Each example uses fakes for a "{}display"{} module, created from a \doxylink{display_8h}{display.\+h} file with {\ttfamily \#include "{}mock\+\_\+display.\+h"{}}. The {\ttfamily \doxylink{display_8h}{display.\+h}} file must exist and must contain the prototypes for the functions to be faked.\hypertarget{README.md_autotoc_md31}{}\doxysubsubsection{\texorpdfstring{Test that a function was called once}{Test that a function was called once}}\label{README.md_autotoc_md31}
